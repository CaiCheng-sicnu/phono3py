The calculation results are written into files. Mostly the data are stored in HDF5 format, therefore how to read the data from HDF5 files is also shown.

Intermediate text files¶

disp_fc3.yaml¶

This is created with -d option. See -d: Create displacements.

disp_fc2.yaml¶

This is created with -d option with --dim_fc2 option. See –dim_fc2: Supercell dimension for 2nd order force constants.

FORCES_FC3¶

This is created with --cf3 option. See –tmax, –tmin, –tstep, –ts: Temperatures.

FORCES_FC2¶

This is created with --cf2 option. See –cf2: Create FORCES_FC2 and –dim_fc2: Supercell dimension for 2nd order force constants.

HDF5 files¶

kappa-*.hdf5¶

See the detail at Details of kappa-*.hdf5 file.

fc3.hdf5¶

Third order force constants (in real space) are stored in .

In phono3py, this is stored in the numpy array dtype='double' and order='C' in the shape of:

(num_atom, num_atom, num_atom, 3, 3, 3)

against . The first three num_atom are the atom indices in supercell corresponding to , , , respectively. The last three elements are the Cartesian coordinates corresponding to , , , respectively.

If you want to import a supercell structure and its fc3, you may suffer from matching its atom index between the supercell and an expected unit cell. This may be easily dealt with by letting phono3py see your supercell as the unit cell (e.g., POSCAR, unitcell.in, etc) and find the unit (primitive) cell using –pa option. For example, let us assume your supercell is the 2x2x2 multiples of your unit cell that has no centring, then your --pa setting will be 1/2 0 0 0 1/2 0 0 1/2 0. If your unit cell is a conventional unit cell and has a centring, e.g., the face centring,

So what you have to set is --pa="0 1/4 1/4  1/4 0 1/4  1/4 1/4 0".

fc2.hdf5¶

Second order force constants are stored in .

In phono3py, this is stored in the numpy array dtype='double' and order='C' in the shape of:

(num_atom, num_atom, 3, 3)

against . More detail is similar to the case for fc3.hdf5.

gamma-*.hdf5¶

Imaginary parts of self energies at harmonic phonon frequencies ( = half linewidths) are stored in THz. See –write_gamma.

gamma_detail-*.hdf5¶

Q-point triplet contributions to imaginary parts of self energies at phonon frequencies (half linewidths) are stored in THz. See –write_gamma_detail.

Simple text file¶

gammas-*.dat¶

Imaginary parts of self energies with respect to frequency are stored in THz. See –ise: Imaginary part of self energy.

jdos-*.dat¶

Joint densities of states are stored in Thz. See –jdos: Joint density of states.
